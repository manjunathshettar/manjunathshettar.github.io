Switch Statements

We looked at the while loop, which is the brother of the for loop. Now let's look at the switch statement, which is a simple alternative to if-else blocks.

Initialize an integer with a value between 10 and 40. Now, let's start the switch statement by typing: switch(variable){. That's the code that lets Java know you're about to dive into a switch statement. Let's create a first case. In the switch block, type in case 10: System.out.println(variable); break;. Looking at this code, we can see two new concepts. Since our variable is an integer, the number after the keyword 'case' is denoting a number as well. It's the equivalent of saying; if(variable == 10) System.out.println(variable);.

The break statement allows the switch statement to quit instantly if the case is true. If-else blocks automatically exit, but you need to include break; if you want to exit out of a switch block. We can have multiple cases in a switch block as well. Underneath the first case, add 3 other cases; case 20: System.out.println("Twenty!"); break;, case 30: System.out.println("Thirty!"); break; , case 40: System.out.println("Fourty!"); break;. We don't need braces here, because the case block reads everything after it until the next case. Change the value of your variable to 40, if it wasn't already. If you implemented the code correctly, "Fourty!" should print out.

Let's try this with booleans now. Initialize a boolean variable with the value of false. Start a new switch statement, putting in the name of your variable in between the switch( //variable goes here ) parameter. Create two cases, one for false and one for true. In the true case, print out true and in the false case, print out false. Run your code and watch the magic work.

How can we make this more simpler? We can add a default value. A default case will run if and only if the other cases don't work. As a result, your default case must be the last case in your switch statement. The code looks something like this; default: //do something;. You do not need a break statement after the default case, since it's the last case. Let's go back to the integer switch statement. Add a default case and change the value of your variable to 24. Run the code and you'll see that nothing appears. Now, add a default case where you print out "Default!". Run the code and you'll see that "Default!" is printed.

Before we finish, we'll look at a common error. Programmers call this case the "cascading case". It occurs when you don't add a break statement at the end of the case. If there's no break statement, the Thread cascades down the switch block, going to every case and doing every action in every case that the conditional holds true. The way to avoid this? Simple, just add break statements at the end of every case!
