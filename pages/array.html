Intro to Arrays

Let's assume that we have a bunch of items. We have apples, oranges and bananas. Now, we want to organize them and seperate them into 3 different bags. Obviously we cannot put an apple into the orange bag. So, each bag will represent a collection of different items.

This is exactly what an array is. An array is a collection of variables of the same data type. Arrays are very useful and used in atleast 90% of all Java algorithms. To initialize an array, we can do one of two things. The first method assigns the array a length, but not the values inside it. int[] arr = new int[length];. This is an integer array, denoted by the int[]. A double array would be the same thing, except with a double[]. The number inside the brackets on the right hand side will denote how many elements can fit inside the array. This value is immutable, which means we cannot change the length of an array after we have assigned it. You can always go back to change the length in the initialization, though.

The second way to create an array is as such; int[] arr = {1, 2, 3, 4};. This method fills in the array with the values passed in the bracket. We don't need to give the array a length because it is already implied by the number of elements being passed. If you already know what elements are going to be in your array, then this is the quickest and most convenient way to fill it in.

If we are creating the array the first way, that means we are creating an array with default values. When any variable is declared but not initialized in a method, Java gives it a default value. For integers and doubles, that value is 0 and 0.0, for booleans it is False, for Strings and characters, the values are empty Strings and characters. So, in our first array, all we have is n zeroes. N being the length of our array. To give each position of the array a value, we can assign it like this (given that the arr is of an integer type): arr[0] = 20;. This initializes the first array spot (remember that Java counts from 0) to 20. We can keep going by doing arr[1] = 20; arr[2] = 30; arr[3] = 5000; ... up until length - 1. The reason for this is because Java counts starting at 0. If we have n elements, that means our last element will be at the index of n-1.

Now that we have the indexes of our array initialized, let's experiment with some stuff. Let's say we want the value of the array at index 3. Put that value into an integer variable. int val = arr[3];. Now, let's try and change the value of the array at index 1 to 80. arr[1] = 80;. The value of the indexes can be changed, but the length of the array is immutable.

The length attribute is a very important to remember. You can manipulate arrays very easily if you remember this one trick. Array-Attribute. What does this mean? Well, remember that for Strings, the length could be received by calling the length() function. For an array, however, the length is an attribute that was pre-determined by you. Instead of using parentheses, we can refer to an attribute by doing arr.attribute. In this case, it's arr.length;. This will return an integer.

Let's make this simpler. Previously, for every index we manually inputted every value into each index. We can use a structure or more specifically, a loop, to make this more efficient. A while loop will do the trick, but a for loop will be far easier to understand. Start by typing for(int i = 0; i < arr.length; i++){}. Notice that our conditional is now different. We are running the for loop as long as i is less than the length of the array, which makes sense because we do not want to go outside the array. There are no values outside the array, so if we were to continue, we would receive an "IndexOutOfBounds Error". Now, let's set each array index to have a value of i. We can do this by typing arr[i] = i;. This means that the first index, 0, will have a value of 0. This will continue to the index of n-1.

What if we wanted to increase an integer/double array in increasing order? Normally, we would have to implement a sorting method and work with recursion, topics you'll learn later. But, recently a new method, arr.sort() was implemented into Java. This is very helpful since it only takes one line to sort an array, as opposed to typing out over 30 lines.
