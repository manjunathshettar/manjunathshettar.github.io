Datatype - Numbers

Numbers

Numbers are fun to play around in Java. We have whole numbers - integers and decimals - doubles. We've already taken a peek at Integers, so let's look at doubles first.

A double is an integer. If asked, you should know that a double takes up 64 bits of data, while an Integer will take 32 bits. A variable of the Double data type is initialized like that of the Integer data type: double variableName = 3.14. A double always has to have a value after its decimal point. Otherwise, Java will think you are assigning an Integer directly to a Double variable. Is that possible?

Actually, it is possible. By declaring a Double object, we can use a method of the Double object to turn an Integers into a decimal. Double var = new Double(100);. In this statement, 100 is our integer. We are casting the Integer into a Double object, giving us a Double. We can also use the .valueOf() method; Double var2 = Double.valueOf(100); will result in the same exact thing. Double.valueOf() is a method that returns the decimal version of the inputted Integer.

Now we know how to go from Integer to Double, now how about Double to Integer? It's fairly simple. Let's say we have this; double doubleVar = 33.444;. Now, let's do this: int newVariable = (int)(doubleVar);. Our code is taking the doubleVar we had initialized previously and forcing it to become an integer. When we do this, the decimal places are truncated and no rounding occurs.

The range of a double variable is ±1.79769313486231570E+308. That's pretty impressive and more than enough for our needs. It's hard to imagine someone using that large/small of a number! We'll get into more methods of the Double object when we are introduced to Strings and their methods.

And finally, the Integer. Possibly the second most used data type in Java, second only to the boolean, and one of the most useful. Integers are essentially doubles, but without the decimals. Integer-Integer operations will always result in an Integer. Likewise, Double-Double operations will always result in a Double. A Double-Integer operation will result in a double, however, as the program tries to promote the Integer into a Double.

Remember libraries from lesson 1? Well, we're getting back into them. At the very top of your file, even before your class declaration, type in: import java.lang.Math;. This imports the Math library to your file. We're only going focus on one method from the Math class, but if you want, you can always go to Oracle's API to view all of the Math methods.

Math.random is in our spotlight. Here is the ideal usage of the .random method: (Math.random()*range)+minimum. The range is simply your maximum subtracted by your minimum and then subtracted again by 1. Your minimum is, well, your minimum. This method will return a double. If you want it to return an Integer, we can cast the whole operation like so; int variable = (int)(Math.random()*range)+minimum;.

Using a print statement, add the integer and double variable together and observe the result. Use Math.random() to create a random integer from the range 2-8.
