Booleans

Can you think of a yes or no question? A yes or no question only has two acceptable outcomes: yes or no. Similarily, Java uses the boolean data type to represent true/false in terms of a variable.

When you declare a boolean; boolean booleanVariable;, the default value will be false. When you print out a boolean, you will either get true or false. We can initialize a boolean like so; boolean bool = true; or boolean bool = false;. Booleans are especially useful when we delve into the world of loops.

Try this; System.out.println(true || false);. Here, we seem to be printing some sort of output of two booleans; of the false and true values. The || in between the booleans represents or. The output of the statement will print either true or false. The or operator will print true if one of the booleans are true. If both are false, 'false' will be printed. If both are true, 'true' will be printed.

Now try this; System.out.println(true && false);. You should've gotten false. This is the and operator. The and operator will only return true if both conditionals are true. If both are false, even if one is false, the statement will return false.

What is short-circuiting? Well, let's look at the or operator. true || false. Since the or operator only requires one of the booleans to be true to return true, we can look at the first variable, true, and see that the operation will return true. However, if the operation is; false || true, we have to look at both variables.

Short-circuiting works with the and operator as well. If we have; false && true, it won't matter what the second variable is, since the and operator requires two trues to return a true. We can look at the first variable and immediately tell that the statement will return false. Now, if we have true && true, we'll have to check both variables to ensure that they are both two.
