While Loops

The while loop is like the for loop's less liked brother. It isn't as efficient as the for loop, but it still gets the job done. Let's look at the syntax

while(i < max){ //do something; i = i +1;}. Let's look at this code. You obviously have to include the 'while' for Java to recognize that you are, in fact, writing a while loop. Inside the parentheses, you need to have a conditional that will be false at one point. In this example, we are looping as long as i is less than max. Since we are incrementing i at the end of the loop (i = i+1;), i will eventually reach the max value. Once it does, the loop breaks off. If you don't make it so that your conditional will eventually become false (or true, in some cases), you will get a StackOverflow error.

You have to initialize and declare i outside of the while loop header. This is where the while loop falls off. The for loop allows you to initialize & declare, increment and condition-check all in one line. In a while loop, you're forced to seperate them.

So, what are the benefits of using a fwhile loop over a for loop? Well, there really aren't any. Sure, if you're limited to only using while loops, a while loop is clearly the better choice. If you want to practice incrementation and conditionals, then a while loop will be good for practice. Other than that, a for loop is far more efficient, quicker and beginner friendly.

There are a lot of common mistakes that even experienced programmers make when they attempt to code while loops. Putting a semicolon like so; while(//some statement); will give you a large amount of errors. Not allowing the conditional to trend to being true/false (depending on your conditional) will give you a StackOverflow error; where the program runs infinitely. Probably the most common error is remembering that you have to declare and initialize the variable before the while loop header.
